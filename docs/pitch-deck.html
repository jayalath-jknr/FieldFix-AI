Root Cause

The Problem Isn't Knowledge. It's Access.

The manual exists. It has every answer. The senior engineer exists. They know every fault.

But neither is accessible in the moment — with dirty hands, under pressure, at the equipment.

Hands Occupied

Can't type, can't scroll. A field technician's primary interface must be voice.

Bad Connectivity

Rural sites, rooftops, basements. Any solution must survive a weak mobile signal.

No Shared Memory

Every repair starts from zero. The fix your colleague did last week is lost unless written down.

The insight: We don't need a smarter chatbot. We need a system that can see what the technician sees, hear them out, and deliver grounded answers — one step at a time — hands-free, in real time.

The Solution

FieldFix AI — Point. Speak. Fix.

Three Core Capabilities
  • See: Point your camera at the equipment. The AI identifies the device and reads the fault state in real time.
  • Speak: Describe the symptom by voice. No typing. Fully hands-free.
  • Guided Fix: Receive one step at a time, read aloud, cited from the actual manual. Interrupt at any moment.
Three Hard AI Constraints
  • ✗ No advice without a manual citation — hallucinations are eliminated by design
  • ✓ One step at a time — waits for confirmation before proceeding
  • ✓ Fully interruptible — technician cuts in, AI stops mid-sentence

How It Works

Architecture Built for the Field

┌─────────────────────────────────┐ │ TECHNICIAN'S PHONE │ │ React PWA · AudioWorklet │ │ Camera · Zustand State │ └───────────┬─────────────────────┘ │ Single Multiplexed WebSocket │ (binary PCM audio + JSON frames) ┌───────────▼─────────────────────┐ │ FastAPI · Cloud Run │ │ ┌───────────────────────────┐ │ │ │ Gemini 2.0 Flash Live │ │ │ │ Bidirectional Audio+Vision│ │ │ └────────────┬──────────────┘ │ │ │ Tool Calls │ │ ┌────────────▼──────────────┐ │ │ │ diagnose_frame │ │ │ │ search_manual (RAG) │ │ │ │ lookup_similar_cases │ │ │ │ save_resolved_case │ │ │ └────────┬──────────┬───────┘ │ └───────────┼──────────┼──────────┘ │ │ ┌───────▼───┐ ┌───▼────────┐ │ Firestore │ │ Cloud GCS │ │ Case DB │ │ Manuals │ └───────────┘ └────────────┘
Key Engineering Decisions
  • Single WebSocket: Binary PCM audio + JSON frames share one connection — critical for weak mobile networks
  • AudioWorklet: Zero-dropout audio capture (vs. deprecated ScriptProcessorNode)
  • Stateless Cloud Run: Horizontally scalable, pay-per-use, scales to zero
  • In-memory cosine similarity RAG: Sub-1ms retrieval, no vector DB needed at MVP scale
Cosine Similarity (RAG)

Query embedding Q vs. chunk embedding C computed in NumPy:

similarity = (Q · C) / (||Q|| × ||C||)

768-dimensional vectors → <1ms per lookup. No shared state between instances.
